# Product Improvement Plan

## Product Goal

把当前“功能集合型”的英语学习工具，逐步打磨成成熟的备考产品：用户打开后知道今天该做什么，学习过程有连续感，阅读、单词、复习、统计形成闭环。

## Current State

已完成的核心方向：

- 今日计划已从入口列表升级为任务台，能根据复习、新词、错题和推荐阅读给出下一步建议。
- 首页、训练中心、背单词、阅读、复习/错题/集合页、统计、测试页已逐步接入共享 `AppLayout`、`Sidebar`、`Topbar` 和移动端 app chrome。
- 测试页白蓝视觉语言已成为全站母版，详见 `test-page-visual-language.md`。
- 训练中心保留 hero、四个主模块、inline 单词/阅读 picker、motivation band、Daily Progress。
- 阅读入口统一回到训练中心阅读 picker；旧独立阅读列表页已删除。
- 阅读正文页已升级为共享 chrome 下的蓝白学习卡结构。
- 手机端单词学习页已改为专属 clean study chrome。
- 登录/注册弹窗已升级为蓝白 SaaS 风格并保留邮箱验证码逻辑。

## Current Priority

继续把“今日学习计划”从轻量推荐升级为完整日任务闭环。

重点：

1. 把阅读完成和考试巩固完成纳入日级进度。
2. 支持每日新词目标配置，例如 10 / 15 / 25。
3. 完成今日任务后给出轻量总结和明日预告。
4. 继续清理 TOEFL/IELTS 选择页和残留旧样式，使其贴近测试页视觉语言。

## Next Product Stages

### 1. 学习目标

目标：让 IELTS / TOEFL / 日常英语目标贯穿首页推荐、词库、阅读和统计。

建议：

- 增加学习目标设置。
- 首页推荐阅读和词库入口优先匹配当前目标。
- 统计页区分目标内进度。

### 2. 阅读闭环

目标：让阅读从“文章 + 题目”升级为可复盘训练。

建议：

- 阅读完成后生成结果页：正确率、错题解析、未掌握词、复习建议。
- 阅读列表显示已完成、正确率、最近练习时间。
- 高亮词弹窗支持“加入今日复习 / 标记已掌握”的明确反馈。

### 3. 复习与错题回收

目标：让错题和到期复习成为用户每天回来的理由。

建议：

- 错题按来源分组：单词测验、填空、拼写、阅读题。
- 连续答对后提示移出错题本。
- 首页突出“今日到期”和“错题回收”。

### 4. 统计产品化

目标：统计不只显示数字，而是回答“我有没有变好”。

建议：

- 增加近 7 天学习趋势、连续学习天数、掌握率变化。
- 区分单词、阅读、考试巩固三个维度。
- 首页任务完成后突出今天新增贡献。

### 5. 新手体验与空状态

目标：首次打开时减少陌生感，让用户快速完成第一次有效学习。

建议：

- 未登录、无学习记录时直接推荐一组新词和一篇短阅读。
- 空状态给出下一步按钮，不只显示“暂无数据”。
- 登录同步入口保留，但不要压过学习主任务。

## Content Rule

扩展 IELTS / TOEFL 阅读和词库时，继续使用“公开事实资料 + 自写文章/题目/解析”的路线，不搬运真题全文。阅读 CSV 固定 `examType`、`level`、`tags`、`questions` 和解析字段，避免字段漂移。

## Acceptance

- 首页 3 秒内能看懂今天推荐做什么。
- 用户能一键进入复习、新词、阅读或考试巩固。
- 桌面端共享 sidebar/topbar，不恢复页面本地 chrome。
- 移动端不卡片挤压、不文字溢出、不横向滚动。
- 训练中心不出现搜索栏，通过四个模块和 inline picker 进入学习。
- 阅读入口必须回到训练中心阅读面板。
- 测试入口必须进入统一考试选择页，全范围随机使用完整词库数量。
- 手机端单词学习页不显示通用 mobile topbar/bottom nav，也不显示桌面状态侧栏。
- 生成图片只作装饰，不能替代文字、按钮、统计、进度或导航。
- `npm run lint` 和 `npm run build` 通过。
