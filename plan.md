# Product Improvement Plan

## 产品目标

把当前“功能集合型”的英语学习工具，逐步打磨成更成熟的备考产品：用户打开后知道今天该做什么，学习过程有连续感，阅读、单词、复习、统计形成闭环，而不是每个模块各自独立。

## 当前优先级

### 1. 今日学习计划 / 首页任务流

状态：进行中。

目标：
- 让首页从“入口列表”升级为“今天先做什么”的任务台。
- 根据用户当前状态自动推荐下一步：到期复习、新词学习、考试阅读、考试巩固。
- 保留全部训练入口，但降低它的主视觉优先级。

已完成：
- `StudyHub` 新增“今日学习计划”卡片。
- 接入今日已学词数、今日已掌握词数、到期复习数、错题数和推荐阅读。
- 推荐阅读按钮会直接进入具体阅读文章，而不是只打开阅读列表。
- 新卡片已补浅色、深色和移动端样式。
- 手机端首页已改为 app 化布局：顶部头像/标题/操作按钮、浮动底部导航、渐变 Hero、状态卡、统计条。
- 首页/训练中心已加入深色/浅色模式切换按钮，并复用现有 `ThemeProvider` 和本地主题存储。
- 训练中心四个入口卡已统一为真实 ModuleCard + 装饰 PNG 资产，不再使用 emoji 作为生产图标。
- 手机端“背单词”现在先展开并滚动到原有词库类型选择区，再由用户选择全部、TOEFL、IELTS 或其它分类。
- 从深层滚动位置切换到 IELTS/TOEFL 等新视图时，页面会自动回到顶部。
- 训练中心桌面端已按现代 English learning app 风格重构为蓝白黄视觉：蓝色 hero banner、白色轻卡、蓝黄 3D 学习图标、强蓝侧边栏选中态、品牌化 Aa 登录入口。
- 训练中心“做阅读”现在和“背单词”一致，先展开 inline 分类面板：阅读按等级展开，单词按词库分类展开，避免跳回旧阅读列表。
- 左侧/移动端“阅读”入口现在也回到训练中心并自动展开阅读面板，不再进入独立阅读列表页。
- 阅读等级卡和文章列表已抽为共享 `ReadingPickerContent`，训练中心和所有阅读入口调用同一套组件。
- 阅读正文页已改为与背单词一致的学习卡方向：共享 `AppLayout` chrome、难词掌握目标条、大阅读卡、桌面右侧阅读状态、底部真实操作按钮，并保留翻译、阅读题、点词弹窗和同步逻辑。
- 旧独立 `ReadingListView.jsx` 已删除；老单词 `CategorySelector.jsx`、老 `components/Progress.jsx` 已删除，当前单词卡重命名为 `WordCard.jsx`。
- 词库分类区已从旧按钮改为紧凑 `ModuleCard`：全部单词 / 日常常用 / 四级核心 / 六级核心 / 托福词汇 / 雅思词汇，并使用对应蓝黄 3D 图标。
- 训练中心底部统计已统一为 Daily Progress 组件，三个 stat 对齐展示，数字和标签保持真实组件。
- 今日计划、训练中心、背单词桌面端已统一共享 `Sidebar` / `Topbar` 外壳，顶部右侧固定为日历、主题、通知、`Aa` 账号。
- 训练中心已移除搜索栏，保持 hero -> 四个训练模块 -> inline 分类面板的清晰路径。
- 手机端单词学习页已改为极简 flashcard 学习界面：返回、标题、进度胶囊、今日目标条、单词卡、底部三按钮，不显示学习进度/连续打卡/坚持每天进步侧栏。
- 考试练习页已统一为共享 `AppLayout`：左侧“测试”和训练中心“做测试”进入同一选择页；全范围随机加载完整 5399 词；四种内部题型使用参考图方向的考试控制条、细进度条和居中答题卡。
- 测试页配色和卡片节奏已确定为后续全站视觉母版：白色大面板、浅蓝背景、强蓝选中态、柔和语义色卡片、透明 3D 模式图标，规范记录在 `test-page-visual-language.md`。
- 共享侧边栏已加入“连续学习”卡，放在桌面 sidebar 底部；考试控制条不再显示“已同步”，账号/登录状态由全局账号 chip 表达。
- 阅读选择面板、阅读正文页和统计页已继续同步到测试页视觉语言：白色/浅蓝面板、强蓝主状态、柔和语义色卡、真实 DOM 图表/进度/按钮，并继续复用共享 `Sidebar` / `Topbar`。
- 计划状态卡和统计卡已移除 emoji 生产图标用法，改为透明 PNG 装饰资产或真实 CSS/DOM 图形。
- 今日复习、错题本、已学习单词和已掌握单词集合页已统一为共享 `AppLayout`：顶部只保留紧凑进度条，工具栏放入内容面板，空状态使用大号透明 PNG 装饰和柔和蓝紫面板，单词卡/按钮/搜索/进度保持真实组件。
- 手机端训练中心已做轻量产品化收紧：hero 从大营销横幅改为紧凑入口卡，四个模块卡统一高度/标题/文案/插画/箭头位置，底部导航和顶部导航保持不变。
- 登录/注册弹窗已按蓝白 SaaS 风格重构：圆角白色 modal、两张真实统计卡、分段登录/注册、带图标输入框、蓝色验证码按钮，并保留原邮箱验证码逻辑。

后续可增强：
- 把阅读完成、考试巩固完成也纳入日级进度，而不是只统计复习和新词。
- 允许用户自定义每日新词目标，例如 10 / 15 / 25。
- 为“今日任务完成”加入轻量总结和明日预告。
- 将 TOEFL/IELTS 列表页继续同步到测试页视觉语言，减少旧样式残留。

### 2. 学习路径与考试目标

目标：
- 让用户选择 IELTS / TOEFL / 通用词汇目标后，首页、词库、阅读和考试练习都围绕该目标组织。
- 减少用户在多个入口里反复思考“我该点哪一个”。

建议实现：
- 增加学习目标设置：IELTS、TOEFL、日常英语。
- 首页推荐阅读和词库入口优先匹配当前目标。
- 统计页区分目标内进度，例如 IELTS 已学、IELTS 已掌握、阅读完成数。

### 3. 阅读练习闭环

目标：
- 让阅读从“文章 + 题目”升级为“可复盘的阅读训练”。

建议实现：
- 阅读完成后生成结果页：正确率、错题解析、文章内未掌握词、建议加入复习的词。
- 记录阅读完成状态，阅读列表显示已完成、正确率、最近练习时间。
- 高亮词弹窗里加入“加入今日复习 / 标记已掌握”的明确反馈。

### 4. 复习与错题回收

目标：
- 让错题和到期复习成为用户每天回来学习的理由。

建议实现：
- 首页展示“今日到期”和“错题回收”两个明确状态。
- 错题本按错误来源分组：单词测验、填空、拼写、阅读题。
- 对连续答对的错题自动提示移出错题本。

### 5. 统计页产品化

目标：
- 统计不只显示数字，而是回答“我有没有变好”。

建议实现：
- 增加近 7 天学习趋势、连续学习天数、掌握率变化。
- 区分单词、阅读、考试巩固三个维度。
- 首页任务完成后跳到统计页时，突出今天新增贡献。

### 6. 新手体验与空状态

目标：
- 首次打开时减少陌生感，让用户快速完成第一次有效学习。

建议实现：
- 未登录、无学习记录时，首页直接推荐一组新词和一篇短阅读。
- 空状态不要只写“暂无数据”，要给一个下一步按钮。
- 登录同步入口保留，但不要压过学习主任务。

### 7. 内容扩展规范

目标：
- 长期扩充 IELTS / TOEFL 阅读和词库时保持合法、稳定、可维护。

建议实现：
- 继续使用“公开事实资料 + 自写文章/题目/解析”的路线，不搬运真题全文。
- 固定阅读 CSV 的题目 JSON 模板，避免字段漂移。
- 每批新增阅读都带 examType、level、tags、questions 和解析。

## 验收口径

- 首页打开后 3 秒内能看懂今天推荐做什么。
- 用户能从首页一键进入复习、新词、阅读或考试巩固。
- 深色模式不出现白色刺眼卡片或浅色文字看不清。
- 移动端首页不卡片挤压、不文字溢出。
- 手机端模块卡不使用 emoji，小图标在右上，按钮触控高度至少 48px。
- 训练中心桌面端不卡片/插画/文字重叠，分类面板无横向溢出。
- 训练中心不出现搜索栏；用户通过四个模块卡和 inline 分类面板进入学习。
- 点击导航“阅读”必须回到训练中心阅读面板；不得出现独立旧阅读列表页。
- 阅读文章选择应为纵向文章行，不使用横排 ModuleCard。
- 统计页必须使用共享 AppLayout，不得恢复旧局部顶部栏；统计数字、趋势条和卡片必须是真实组件。
- 复习 / 错题 / 已学习 / 已掌握集合页必须使用共享 AppLayout；全局 topbar 只保留标准槽位，页面内只保留紧凑进度条和内容面板工具栏。
- 登录/注册弹窗必须保留真实表单和邮箱验证码流程；统计数字、输入框、分段控件、错误/成功状态都必须是真实 DOM，不得用静态截图替代。
- 点击导航“测试”和训练中心“做测试”必须进入同一考试选择页；全范围随机应显示完整词库数量并能进入四种题型。
- 考试内部页必须保留共享 sidebar/topbar，题型内部不得恢复旧局部 topbar；控制条不得显示“已同步”。
- 手机端单词学习页不显示通用 mobile topbar/bottom nav，不显示桌面学习状态侧栏，底部三个操作按钮横向排列且不被安全区遮挡。
- 训练中心生成图片只作为装饰资产，不能替代按钮、文字、词数、统计数字或进度。
- `npm run lint` 和 `npm run build` 通过。
